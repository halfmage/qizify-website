# Sales kit: custom learning software for DACH Bildungsträger

Go-to-market kit for selling LearnSlice custom software engineering (AI learning coach, LMS, learning platforms) to training providers in the DACH market. Built from market research, June 2026.

## Files
- `icp.md`, need-ranked segments + funding path + buying committee
- `target-list.md`, how to build the account list + worked example accounts
- `linkedin-and-inbound-playbook.md`, UWG-safe outreach motion (no cold email)
- `objections.md`, objection handling
- `qualification.md`, qualifying inbound leads from the new page
- `lead-magnet-bfsg-azav-checklist.md`, gated inbound asset that powers the motion

## The one thing to remember
German training funding (AZAV/Bildungsgutschein via the Bundesagentur für Arbeit, DRV for reha) is per-participant operating reimbursement, not capex. It cannot fund a bespoke software project, only a proportional slice of a software license as Sachkosten. **So we sell a managed per-seat subscription (opex), not a capex build.** That packaging converts the highest-need, capex-poor segments into buyers, and ESF Plus co-finances the rest.

## What must be confirmed before this goes live (see plan Part 7)
1. Sachkosten-recoverability of per-seat software cost with an AZAV Zertifizierer (linchpin of the funding argument).
2. The legal claims (BFSG dates, AZAV 1 Jul 2025 synchronous rule, ESF+ priorities) against primary sources before they appear in customer-facing material.
3. Which compliance features (synchronous logging, attendance/Verbleib reporting, WCAG 2.1 AA) exist today vs. are custom-built, so we never overclaim. Soften "AZAV-konform" to "AZAV-unterstützend".

Scope: **Germany-first.** AZAV and BFSG are German law; Austria (AMS, Ö-Cert) and Switzerland (eduQua, SBFI) need re-framed messaging later.
